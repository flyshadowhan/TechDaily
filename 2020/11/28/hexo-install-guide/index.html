Hexo@Github Pages 搭建指南

本文字数： 3.4k 阅读时长 ≈ 3 分钟
初次用Hexo在Github上搭建站点,参考了不少人的攻略.在这里记录一下整个站点的搭建过程.

前言

初次用Hexo在Github上搭建站点,参考了不少人的攻略.在这里记录一下整个站点的搭建过程.

Ok,here we go~

准备环境

Hexo是高效的静态站点生成框架,它基于Node.js.Hexo对OS没有特别要求,windows或者linux均可,本文基于Ubuntu 18.04.

node环境安装

先刷新apt索引

安装nodejs和npm

注意:Hexo对nodejs和npm有版本要求.请确认nodejs版本>=10.13.0,npm>=3.5.2

国内访问npm库速度较慢,设置npm镜像加速

git安装

安装git,用来上传博客至Github.

完成后可以看到git版本

至此运行环境的准备就绪.

Hexo安装

安装Hexo

完成后,可以查看Hexo版本信息

Hexo使用

接下来就是Hexo的配置和使用.

Hexo初始化

在当前目录下输入hexo init,Hexo在此目录进行一个初始化,或者初始化指定目录hexo init [path].我的博客目录设定为/blog,下文相同.

此步骤运行时间较长,并且没有相应进度提示,耐心等待.从日志来看,时间主要花费在了Github上拉取Hexo环境.
当看到日志中的”Start blogging with Hexo!”说明Hexo已经可以正常运行了.

Hexo运行

在当前目录/blog下输入hexo new post "hello-world",Hexo会创建一篇新博客.

通过日志可以看到,所有的markdown文件存放在了/blog/source/_posts/下.

目录保持不变继续输入hexo generate,Hexo会将/blog/source/_posts/下所有markdown文件转换成静态页面.

转换完成后,你可以看到当前目录/blog下多了public目录,所有转换完成的文件都在这里.后续也就是将这个目录发布至Github Pages.

转换完成后,输入hexo server.Hexo服务将会启动.

在浏览器中输入http://localhost:4000或者http://[IP]:4000就可以访问本地的Hexo站点了.输入ctrl+c,退出Hexo服务.
上面三个命令包含了创建博客,生成页面,启动本地预览的一个过程.更多命令的详细说明可以参考Hexo中文官网Hexo命令.

发布至Github Pages

刚刚只是在本地预览了博客,下面就来说明如何将博客发布至Github Pages.Github的账号注册在这里就略过,我们直接进入发布环节.

安装Hexo发布插件

安装hexo-deployer-git

这样,Hexo具备提交Github的功能.

Github设置

登录GitHub,点击New repository,创建一个新的repository.:joy:

在repository name里输入项目名,即未来访问的博客名.有的参考资料提到这里的名字一定加github.io后缀.从使用情况看,加与不加都能正常使用,所以看你喜欢.项目勾选Public,才能被博客外部访问.记得勾选Add a README file,后续有用.最后点击create repository.

项目创建后,点击settings.

这里显示项目的各种设定.下拉找到其中GitHub Pages的设定.因为勾选了创建README文件,GitHub自动创建了一个分支main.对,因为今年的BLM运动曾经的MASTER被改成了main…
选择branch: main, /(root),点击Save.保存完成后,Github会给出一个外部的访问地址,将这个地址保存起来,这个就是博客的访问地址.

注意:有的参考资料提到选择一个theme,完全没必要.Hexo的主题并不是通过这里设定的.

修改_config.yml

打开/blog目录下_config.yml,找到其中的deploy部份.repository填刚刚创建的项目地址,branch填main分支.

注意,:后面有空格

提交Github

到这里,已经到了最后一个环节.在/blog下执行hexo generate后输入hexo deploy

Hexo会自动调用git,提交/public目录中的所有文件至GitHub Pages.一切顺利的话,你的博客就可以通过刚刚GitHub Pages设定中给的地址访问了.

总结

Hexo安装和配置并不复杂,只是其中的环节有些多.细心一些,可以少走很多弯路,快速完成.下一篇会是如何通过集成Github Actions来自动发布Github Pages.
